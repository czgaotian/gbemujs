# Game Boy CPU 本地参考

本文件把原先的 CPU 参考链接整理为可离线阅读的实现摘要。它面向本仓库的
LR35902 实现；具体的模块分工和代码约定见同目录的
[`CPU_IMPLEMENTATION_GUIDE.md`](CPU_IMPLEMENTATION_GUIDE.md)。

## 处理器与寄存器

Game Boy 使用 Sharp LR35902：8 位数据路径、16 位地址和程序计数器，单速时钟
为 4.194304 MHz。CPU 可直接寻址 64 KiB（`0x0000`–`0xFFFF`）。

可独立访问的 8 位寄存器为 `A`、`F`、`B`、`C`、`D`、`E`、`H`、`L`；其中
`A` 是累加器，`F` 是状态寄存器。它们可组成 `AF`、`BC`、`DE`、`HL` 四个
16 位寄存器对。`PC` 保存下一条指令地址，`SP` 保存栈顶地址。

`F` 的低 4 位始终为零，只有下列位有效：

| 位  | 掩码   | 名称 | 含义                               |
| --- | ------ | ---- | ---------------------------------- |
| 7   | `0x80` | Z    | 结果为零                           |
| 6   | `0x40` | N    | 最近一次算术运算为减法             |
| 5   | `0x20` | H    | 第 3 位到第 4 位发生半进位或半借位 |
| 4   | `0x10` | C    | 第 7 位发生进位或借位              |

实现中写入 `AF` 或 `F` 时必须掩掉低半字节：`F &= 0xF0`。`Z`、`N`、`H`、`C`
并非所有指令都会更新；保留的标志必须保持原值。

## 标志位判定

- 8 位加法：`Z = result === 0`，`N = 0`，`H` 在 `(a & 0x0f) + (b & 0x0f) + carry`
  超过 `0x0f` 时置位，`C` 在完整和超过 `0xff` 时置位。
- 8 位减法：`Z = result === 0`，`N = 1`，`H` 在低半字节需要借位时置位，`C` 在
  `a < b + borrow` 时置位。
- 16 位 `ADD HL, rr`：`Z` 保留，`N = 0`；半进位测试低 12 位，进位测试完整
  16 位和。
- `INC` 不改变 `C`；`DEC` 不改变 `C`，但会置 `N`。`CP` 的标志与 `SUB` 相同，
  但不写回 `A`。
- `AND` 固定 `H = 1, N = C = 0`；`OR` 与 `XOR` 固定 `N = H = C = 0`。
- `DAA` 依赖先前的 `N`、`H`、`C` 来修正 BCD 结果；执行后 `H = 0`，`N` 保留。

## 指令编码结构

共有 256 个基础操作码和 256 个 `0xCB` 前缀操作码。基础操作码常可按
`x = opcode >> 6`、`y = (opcode >> 3) & 7`、`z = opcode & 7` 解码：

| 区块 | 范围          | 主要内容                                             |
| ---- | ------------- | ---------------------------------------------------- |
| 0    | `0x00`–`0x3F` | 跳转、16 位加载和算术、`INC`/`DEC`、立即数加载、旋转 |
| 1    | `0x40`–`0x7F` | `LD r, r`；`0x76` 是 `HALT`，不是一次加载            |
| 2    | `0x80`–`0xBF` | `ADD/ADC/SUB/SBC/AND/XOR/OR/CP A, r`                 |
| 3    | `0xC0`–`0xFF` | 返回、调用、重启、栈、立即数算术、`LDH`、`DI/EI`     |

8 位寄存器字段的编码顺序为 `B, C, D, E, H, L, (HL), A`。因此使用 `(HL)` 的
指令会额外访问一次内存，时序通常比寄存器版本长。

`0xCB` 前缀后的操作码也按字段排列：低 3 位选择目标
`B, C, D, E, H, L, (HL), A`；高位选择操作。

| 范围            | 操作                                               |
| --------------- | -------------------------------------------------- |
| `CB 00`–`CB 3F` | `RLC/RRC/RL/RR/SLA/SRA/SWAP/SRL`                   |
| `CB 40`–`CB 7F` | `BIT b, r`；不修改目标，`C` 保留，`H = 1`，`N = 0` |
| `CB 80`–`CB BF` | `RES b, r`                                         |
| `CB C0`–`CB FF` | `SET b, r`                                         |

`RLCA`、`RRCA`、`RLA`、`RRA` 是累加器专用旋转，执行后 `Z` 总为零；CB 前缀
旋转则按结果设置 `Z`。`SRA` 保留 bit 7，`SRL` 补零，`SWAP` 交换高低半字节。

## 控制流、栈与中断

立即数使用小端序：读取 16 位值时先取低字节、再取高字节。相对跳转的位移是
有符号 8 位值，基址是读取位移之后的 `PC`。

- `JR cc, e8` 与 `JP cc, a16`：条件不成立时仍消耗操作数，但不跳转。
- `CALL a16`：先读取目标，再将返回地址（当前 `PC`）以高字节、低字节的顺序压栈。
- `RET`：以低字节、再高字节的顺序出栈到 `PC`。
- `PUSH rr` 先递减 `SP` 并写高字节，再递减并写低字节；`POP AF` 必须清除 `F`
  的低 4 位。
- `RST n` 等价于调用固定向量：`00, 08, 10, 18, 20, 28, 30, 38`。
- `DI` 立即关闭 `IME`；`EI` 在**下一条指令完成后**才开启 `IME`。`RETI` 返回并立即
  开启 `IME`。
- `HALT` 在 `IME = 0` 且存在已请求、已使能中断时触发 HALT bug：下一次取指不递增
  `PC`。这是兼容测试 ROM 的必要边界情况。

## 与 Z80 的关系

LR35902 与 Z80 相近，但并不是 Z80 的二进制兼容子集。实现时不能直接复用 Z80
解码表：它没有 `IX/IY`、备用寄存器、`IN/OUT`、`EX`、`DD/FD/ED` 前缀，也没有
符号/奇偶标志相关条件。I/O 使用内存映射，`LDH` 专门覆盖 `0xFF00 + a8` 地址；
它还加入了 `STOP`、`SWAP`、自动递增/递减的 `LDI/LDD`（当前命名通常为
`LD (HL+)/(HL-), A` 与反向形式）。

## 时序与实现检查表

本项目以 machine cycle（4 个时钟周期）为 `tick` 单位。指令执行后应将实际消耗的
tick 交给系统时钟；条件分支、`(HL)` 操作数、`CALL/RET` 等都有可变时序。

实现或审查一条指令时，至少检查：

1. 取指后 `PC` 的位置，以及立即数的字节序；
2. 寄存器或内存的读写顺序；
3. 每一位标志是设置、清除、保留还是固定为零；
4. 掩码后的 8/16 位结果；
5. 基础与条件成立/不成立时的 tick 数；
6. `HALT`、`STOP`、`EI` 延迟和中断边界行为。
